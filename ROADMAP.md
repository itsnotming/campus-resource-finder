# Roadmap

Campus Resource Finder is currently in the planning and early design stage. This roadmap describes the major goals for turning the project into a useful open-source tool for students and campus communities.

The timeline below is flexible. Contributors are welcome to suggest changes, add ideas, or help define future priorities.

## Phase 1: Project Planning and Documentation

Goal: Create a clear foundation for future contributors.

Planned work:

- Define the main problem the project solves
- Identify intended users and use cases
- Create project documentation
- Add contribution guidelines
- Create beginner-friendly Issues
- Decide what types of campus resources should be included first

Possible contributor tasks:

- Improve the README
- Add examples of campus resources
- Suggest categories for organizing resources
- Review documentation for clarity and accessibility

## Phase 2: Resource Data Structure

Goal: Design a simple way to store and organize campus resource information.

Planned work:

- Define required fields for each resource
- Create sample data for campus resources
- Organize resources by category
- Include contact information, location, hours, and eligibility notes
- Add guidance for keeping resource information accurate

Possible contributor tasks:

- Create a sample resource list
- Suggest useful data fields
- Research common campus support services
- Write guidelines for submitting new resources

## Phase 3: Search and Browse Prototype

Goal: Build the first simple version of the resource finder.

Planned work:

- Create a basic web interface
- Add keyword search
- Add category filters
- Display resource details clearly
- Make the interface usable on desktop and mobile

Possible contributor tasks:

- Design a resource card layout
- Create wireframes or mockups
- Help choose a beginner-friendly technology stack
- Build placeholder UI components

## Phase 4: Accessibility and Student Experience

Goal: Make the project useful for a wide range of students.

Planned work:

- Improve keyboard navigation
- Use readable text and accessible color contrast
- Support multiple languages
- Add clear descriptions for services
- Consider the needs of new students, international students, and students with disabilities

Possible contributor tasks:

- Review the interface for accessibility issues
- Translate sample resource descriptions
- Improve wording for clarity
- Test the project with different screen sizes

## Phase 5: Community Contribution Workflow

Goal: Make it easy for people to keep campus resource information up to date.

Planned work:

- Create a resource submission template
- Add an Issue template for resource updates
- Add a process for reviewing submitted changes
- Document how student organizations or campus offices can contribute
- Create clear labels for beginner-friendly and documentation Issues

Possible contributor tasks:

- Write resource update guidelines
- Create GitHub Issue templates
- Suggest labels for project maintenance
- Help review submitted resource information

## Future Ideas

These ideas are not required for the first version, but they could be explored later:

- Map view for campus resources
- Calendar integration for office hours or events
- Saved favorite resources
- University-specific resource collections
- Admin review dashboard
- Public API for campus resource data
- Offline-friendly access for important resources

## Current Status

The project is not yet a working application. The current focus is on planning, documentation, sample data, and contributor experience.

New contributors can help immediately by improving documentation, suggesting campus resource categories, creating sample resource entries, and helping define the first prototype.
